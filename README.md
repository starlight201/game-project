

# DuckLife 小鸭生活模拟系统

## 项目概述
DuckLife 是一个集成了多种趣味功能的Java桌面应用程序，是《现代软件工程课程设计》的课程项目。该系统采用模块化设计，核心功能包括鸭子抢红包游戏、小鸭装扮系统、AI智能对话、代码统计工具以及学生考勤管理，并通过统一的主界面和数据库进行集成。

## 主要功能
1.  **鸭子抢红包游戏**
    *   **核心玩法**：控制坦克在规定时间（60秒）内移动，碰撞并收集屏幕上随机出现的红包。
    *   **游戏特性**：
        *   采用**抽象工厂模式**生成四种形状（方形、圆形、三角形、星形）、三种尺寸的红包。
        *   实现基于物理引擎的红包移动算法，包含边界碰撞和随机转向。
        *   使用 `Area` 类进行坦克与复杂形状（如星形）红包之间的**精确碰撞检测**。
        *   游戏金额实时保存至数据库。

2.  **装扮鸭子系统**
    *   **核心玩法**：为用户的小鸭角色搭配各种装饰品，并生成定制图片。
    *   **系统特性**：
        *   采用**装饰器模式**设计，支持帽子、眼镜、围巾、领带、手杖等5类装饰品的动态叠加与兼容性检查。
        *   实现基于 `BufferedImage` 和 `Graphics2D` 的多层PNG图像合成与渲染，支持透明通道处理。
        *   内置虚拟商店，用户可使用游戏获得的金币购买装扮，所有资产和搭配方案均持久化存储于数据库。

3.  **AI对话模块**
    *   **核心功能**：集成百度千帆搜索API，实现智能问答。
    *   **模块特性**：
        *   封装了百度API的HTTP请求与JSON解析，具备重试机制。
        *   通过 `SwingWorker` 实现异步搜索，避免图形界面卡顿。
        *   提供富文本（HTML）结果展示，并支持超链接直接跳转。

4.  **主界面集成**
    *   **统一导航**：提供 `GameMainMenu` 主窗口，集中管理所有子系统（红包游戏、装扮、AI对话、代码统计、考勤）的入口。
    *   **架构特性**：采用事件监听与回调机制，管理子窗口的生命周期，确保界面状态同步和视觉风格统一。

5.  **数据库集成**
    *   **数据持久化**：使用MySQL数据库，通过连接池管理，为所有模块提供统一的数据层支持。
    *   **核心数据表**：
        *   **用户与资产**：`user`（用户资金）、`shop_items`（商店）、`user_owned_items`（用户拥有物品）。
        *   **装扮系统**：`user_current_outfit`（当前穿着）、`user_saved_outfits`（保存的搭配）。
        *   **考勤系统**：`students`（学生信息）、`attendance_records`（考勤记录）。
        *   **游戏系统**：相关表与用户资金表关联，实现金额同步。

## 技术栈
- **编程语言**: Java
- **图形界面**: Swing
- **数据库**: MySQL
- **API集成**: 百度千帆大模型
- **设计模式**: 抽象工厂模式、装饰器模式、单例模式
- **事务管理**: Spring `@Transactional` 注解（根据报告提及）
- **并发控制**: `synchronized`、`ReentrantLock`

## 快速开始
### 环境要求
1.  Java JDK 8 或以上版本。
2.  MySQL 5.7 或以上版本。
3.  有效的百度千帆API密钥（用于AI对话模块）。

### 运行步骤
1.  **克隆项目**：将项目代码导入IDE（如IntelliJ IDEA）。
2.  **数据库配置**：在 `DatabaseConfig` 类中配置您的MySQL数据库连接信息（URL，用户名，密码）。
3.  **初始化数据库**：运行项目SQL脚本，创建所需的数据表。
4.  **API配置**：在 `SearchService` 类中配置您的百度千帆API密钥。
5.  **运行项目**：启动 `GameMainMenu` 主类，即可进入系统主界面。

## 项目结构 (简要)
```
src/
├── main/
│   ├── game/              # 红包游戏核心逻辑（工厂、红包实体、碰撞检测）
│   ├── dressup/           # 装扮系统（装饰器、图片生成器、管理类）
│   ├── ai/                # AI对话模块（API服务、GUI）
│   ├── menu/              # 主界面集成
│   ├── dao/               # 数据访问对象（DatabaseConfig, RedPacketDAO等）
│   └── utils/             # 工具类
└── resources/             # 图片、配置文件等
```
